# Инструкции для AI-агента

**Перед любой задачей прочитай:** [docs/AI-CONTEXT.md](docs/AI-CONTEXT.md)

Там: репозиторий, прод URL, деплой, Blob, команды `sync:production-offer`, структура лендинга и админки.

## Автономный режим

Пользователь хочет, чтобы агент **сам доводил задачи до конца** — без просьб нажать Run.

### Поведение агента

- Сам запускай команды, билд, диагностику, commit и push на `main` после задачи
- Не пиши «выполните у себя» — делай через терминал
- **Деплой по умолчанию** — не жди «задеплой», если пользователь не сказал «без деплоя» / «только локально»

### Настройка Cursor (один раз)

Чтобы команды шли без кнопки Run:

1. **Cursor Settings → Agents → Run Mode**
2. Выбери один из режимов:
   - **Run Everything** — ноль подтверждений (максимум автономии)
   - **Auto-review** — умный режим + `permissions.json`
   - **Allowlist** — команды из allowlist выполняются сразу

3. Перезапуск не нужен — `permissions.json` подхватывается автоматически.

### Где лежит конфиг

| Уровень | Файлы |
|---------|-------|
| **Глобально (все проекты)** | `C:\Users\user\.cursor\permissions.json`, `C:\Users\user\.cursor\rules\autonomous-agent.mdc` |
| **Этот проект** | `.cursor/permissions.json`, `.cursor/rules/autonomous-agent.mdc` |

Оба уровня **складываются** — глобальные правила работают везде, проектные дополняют их.
